## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

Create a `.env` file in the root of the `api-nestjs` project with the following variables:

```env
# URL of the Python backend service
PYTHON_SERVICE_URL=http://your-python-backend-service-url:8000/process

# Port where the NestJS API will run
# Default is 3001 if not specified and not overridden by Docker
PORT=3001

# Environment (development, production, test, etc.)
# This affects how run.sh starts the application inside Docker (dev vs prod)
# and how NestJS behaves (e.g. logging, error details).
NODE_ENV=development
```
Ensure you replace `http://your-python-backend-service-url:8000/process` with the actual URL of your Python backend.

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run with Docker

This project includes a `Dockerfile` and a `run.sh` script to facilitate running the application in a Docker container.

1.  **Build the Docker image:**
    Open a terminal in the `api-nestjs` directory and run:
    ```bash
    docker build -t api-nestjs .
    ```

2.  **Run the Docker container:**
    After building the image, you can run the container using:
    ```bash
    docker run -p 3001:3001 --env-file .env api-nestjs
    ```
    Or, if you want to specify the `NODE_ENV` directly (e.g., for production):
    ```bash
    docker run -p 3001:3001 -e NODE_ENV=prod --env-file .env api-nestjs
    ```

    Explanation:
    - `-p 3001:3001`: Maps port 3001 of the container to port 3001 on your host machine. The application inside Docker listens on port 3001 as exposed in the `Dockerfile`.
    - `--env-file .env`: Loads environment variables from your `.env` file into the container.
    - `-e NODE_ENV=prod`: Sets the `NODE_ENV` environment variable to `prod`. The `run.sh` script uses this to start the application in production mode (`npm run start:prod`). If omitted or set to a different value (like `dev` or `development`), it will run in development mode (`npm run start:dev`).
    - `api-nestjs`: The name of the image to run.

## Deployment

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

If you are looking for a cloud-based platform to deploy your NestJS application, check out [Mau](https://mau.nestjs.com), our official platform for deploying NestJS applications on AWS. Mau makes deployment straightforward and fast, requiring just a few simple steps:

```bash
$ npm install -g @nestjs/mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.
